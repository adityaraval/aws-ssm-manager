Prerequisites

Before getting started, make sure you have these tools installed.

AWS CLI v2

Required for authentication and session management.

brew install awscli Installation guide

Session Manager Plugin

Required to establish SSM sessions.

brew install --cask session-manager-plugin Installation guide

AWS Credentials

Configure named profiles with your AWS credentials.

aws configure --profile my-profile

Supports IAM users, roles, and SSO authentication.

IAM Permissions

Your IAM user/role needs SSM permissions.

ssm:StartSession
ssm:TerminateSession

Plus EC2 describe permissions for instance lookup.

EC2 Instance Requirements

  • SSM Agent installed and running (pre-installed on Amazon Linux 2, Ubuntu 20.04+)
  • IAM instance profile with AmazonSSMManagedInstanceCore policy
  • Outbound internet access or VPC endpoints for SSM

How it works

Three simple steps to connect to your private AWS resources.

1

Select your AWS profile

SSM Manager automatically detects your AWS CLI profiles. Just pick one from the dropdown.

2

Configure your connection

Choose your service type, enter your bastion EC2 instance ID and the target service endpoint. Ports are auto-filled.

3

Connect and save

Click "Start Session" and you're in. Save it for one-click access next time. View real-time logs in the terminal panel.

macOS: "Damaged and can't be opened"?

This is a Gatekeeper quarantine flag macOS sets on browser downloads — not actual damage. The app is intact. Run this once in Terminal to clear it:

xattr -cr "/Applications/AWS SSM Manager.app"

This only removes the quarantine flag for this app — it doesn't change any system-wide security settings.

Why you can trust this app

  • Fully open source. Every line of code is publicly readable on GitHub — nothing is obfuscated or minified.
  • No outbound network calls. The app makes no requests of its own — all traffic goes through the AWS CLI you already have installed.
  • No telemetry or analytics. Data is stored only in your local browser storage. Nothing is sent anywhere.
  • Verifiable builds. SHA-256 checksums are published with every release. Or build from source yourself.
